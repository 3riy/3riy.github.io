Monitoring Processes

1. ps

Provides a static snapshot of current processes. Use 'aux' to see all users' processes.

ps aux | grep nginx

2. top

Displays an interactive, real-time list of processes sorted by CPU usage.

top

3. htop

An advanced, color-coded process viewer with mouse support and horizontal scrolling.

htop

4. pgrep

Quickly find the Process ID (PID) of a process based on its name.

pgrep -l sshd

5. pstree

Shows the hierarchy of running processes in a tree-like format.

pstree -p

6. pidof

Returns the exact PID of a running program.

pidof docker

▼ Advanced

7. lsof

Lists open files and the processes that are using them—great for finding port conflicts.

lsof -i :80

8. vmstat

Reports virtual memory statistics, CPU load, and system interrupts.

vmstat 2 5

9. iostat

Monitors system input/output device loading (Disk I/O).

iostat -dx 1

10. strace

Intercepts and records system calls made by a process. High-level debugging tool.

strace -p [pid]

11. watch

Runs a command repeatedly at intervals, highlighting changes.

watch -n 1 "ps -eo pcpu,pmem,comm --sort=-pcpu | head -6"

12. sar

System Activity Reporter. Used to collect and log long-term system performance data.

sar -u 1 3

Managing Processes

1. kill

Sends a signal to a process. Signal 9 (SIGKILL) forces an immediate stop.

kill -9 1234

2. pkill

Kills processes based on their name patterns instead of PIDs.

pkill -u riya chrome

3. killall

Matches process names exactly to terminate every instance.

killall node

4. nice

Runs a command with a modified scheduling priority (niceness).

nice -n 15 ./compilation_script.sh

5. renice

Changes the priority of an already running process.

renice +10 -p 4567

▼ Advanced

6. & (Background Operator)

Runs a task in the background, freeing up the terminal immediately.

./backup.sh &

7. jobs

Lists the background jobs currently managed by your shell.

jobs -l

8. fg / bg

Moves jobs between the foreground and the background.

fg %1
bg %2

9. nohup

Allows a process to continue running even after you disconnect/logout.

nohup python app.py &

10. disown

Removes a background job from the shell's job control, preventing it from closing on exit.

disown -h %1

11. timeout

Executes a command but automatically kills it if it takes too long.

timeout 60s ping google.com

System Information

1. uname

Displays kernel version, architecture, and OS details.

uname -a

2. uptime

Tells you how long the machine has been on and the current load average.

uptime -p

3. free

Shows total, used, and free physical and swap memory.

free -h

4. lscpu

A detailed view of the CPU: cores, threads, virtualization, and flags.

lscpu

5. hostnamectl

Displays or sets the system's network hostname and chassis type.

hostnamectl status

▼ Advanced

6. dmesg

Prints the kernel ring buffer—the first place to look for hardware failures.

dmesg | grep -i error

7. lsmod

Lists currently loaded kernel modules (drivers).

lsmod | head -n 15

8. sysctl

Modify kernel parameters at runtime (e.g., networking tweaks).

sysctl -a | grep "ipv4.conf"

9. dmidecode

Reads the SMBIOS table to show hardware serial numbers and BIOS info.

sudo dmidecode -s system-serial-number

10. journalctl

Query and display logs from systemd-journald.

journalctl -xe --unit=ssh.service

11. lsmem

Lists the ranges of available memory and their online status.

lsmem --summary
